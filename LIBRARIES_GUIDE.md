## 2Ô∏è‚É£ TanStack Query (React Query) - Data Fetching

**Why React Query?**
- ‚úÖ Auto caching & revalidation
- ‚úÖ Background refetching
- ‚úÖ Dedupe requests
- ‚úÖ Retry logic
- ‚úÖ Pagination & infinite scroll

### Setup Provider

```typescript
// app/providers.tsx
'use client';

import { QueryClient, QueryClientProvider } from '@tanstack/react-query';
import { useState } from 'react';

export function Providers({ children }: { children: React.ReactNode }) {
  const [queryClient] = useState(() => new QueryClient({
    defaultOptions: {
      queries: {
        staleTime: 5 * 60 * 1000, // 5 ph√∫t
        cacheTime: 10 * 60 * 1000, // 10 ph√∫t
        retry: 2,
      },
    },
  }));

  return (
    <QueryClientProvider client={queryClient}>
      {children}
    </QueryClientProvider>
  );
}
```

## üéØ Best Practices

### ‚úÖ State Management

1. **Use Zustand for:**
   - Global UI state (modals, sidebar)
   - User preferences (theme, favorites)
   - Filters & selections

2. **Use React Query for:**
   - Server data fetching
   - API caching
   - Mutations (POST/PUT/DELETE)

3. **Use useState for:**
   - Local component state
   - Form inputs
   - Toggle states

### ‚úÖ Performance

```typescript
// ‚úÖ Good Pattern
const favorites = useFavoriteStore(state => state.favorites); // Selective
const { data } = useQuery({ queryKey: ['cars'], ... }); // Auto cache

// ‚ùå Bad Pattern

const [data, setData] = useState([]); // Manual fetch & cache
```
